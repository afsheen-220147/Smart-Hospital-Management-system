─ Doctor must be logged in ✓

─ Code validates within expiry time (3 min) ✓
─ Code one-time use (deleted after match) ✓
─ Max 3 attempts per challenge ✓
─ Challenge auto-expires ✓
─ Server-side validation only ✓
─ No sensitive data in frontend ✓

BACKEND CONSENT SYSTEM (Integrated)
─ Code generation: 2-digit (00-99)
─ Code expiry: 3 minutes
─ Challenge options: 3 shuffled numbers
─ Correct code in options: Always
─ Attempt tracking: Enforced (max 3)
─ Auto-cleanup: Every 60 seconds
─ In-memory storage: No database dependency
─ Timestamp-based: Server-side expiry

🔄 COMPLETE DATA FLOW
═════════════════════════════════════════════════════════════════════════════

PAST HISTORY FLOW
─────────────────

Doctor clicks [View Past History]
         ↓
PastHistoryModal opens
         ↓
Modal receives: patientId, patientName, visitHistory
         ↓
Uses local data: selected.visits from PatientDetails
         ↓
Displays timeline with:
  • All visits from selected.visits array
  • Expandable details
  • Summary cards
         ↓
User can:
  • Expand/collapse entries
  • Download records
  • Print records
  • Close modal

FULL MEDICAL HISTORY FLOW
──────────────────────────

Doctor clicks [View Full Medical History]
         ↓
FullMedicalHistoryModal opens
         ↓
Modal state: hasConsent = false, fullHistory = null
         ↓
Display: Consent screen with DoctorConsent component
         ↓
Doctor clicks [Start Verification]
         ↓
DoctorConsent → GET /api/v1/consent/doctor/challenge/patientId
         ↓
Backend returns:
  {
    challengeId: "unique-id",
    options: ["82", "37", "19"]  // Shuffled, includes correct code
  }
         ↓
Display 3 button options
         ↓
Doctor asks patient for code
Patient checks dashboard → sees code (e.g., "37")
Patient tells doctor code
         ↓
Doctor clicks matching button (e.g., "37")
         ↓
DoctorConsent → POST /api/v1/consent/doctor/verify
  Body: { challengeId: "...", selectedCode: "37" }
         ↓
Backend validates:
  ✓ Challenge exists
  ✓ Not expired
  ✓ Attempts < 3
  ✓ Code matches
         ↓
If valid:
  • setHasConsent = true
  • Delete challenge
         ↓
FullMedicalHistoryModal:
  • Display success banner
  • Fetch full patient data from /patients API
  • Display:
    - Personal Info
    - Medical Conditions
    - Allergies
    - Medications
    - Diagnoses
         ↓
User can:
  • Download full history
  • Print records
  • Close modal
