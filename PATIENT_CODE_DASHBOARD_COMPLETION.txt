🔄 COMPLETE DATA FLOW
═════════════════════════════════════════════════════════════════════════════════

PATIENT SIDE (Dashboard)
─────────────────────────────────────────────────────────────────────────────
1. Patient logs in
   └─ Browser: GET /patient/dashboard

2. Dashboard loads
   └─ PatientConsent component mounts
   └─ Effect hook: useEffect runs

3. Initial code fetch
   └─ API: GET /api/v1/consent/patient/code/{patientId}
   └─ Response: { code: "37" }
   └─ Display: "37" shown in sidebar

4. Interval set
   └─ setInterval every 3 minutes
   └─ Auto-fetch new code

5. After 3 minutes
   └─ Interval triggers
   └─ API: GET /api/v1/consent/patient/code/{patientId}
   └─ Response: { code: "82" } (new code)
   └─ Display: "82" shown (updated)

6. Cycle continues
   └─ Every 3 minutes → New code
   └─ While dashboard is open

7. Patient navigation
   └─ Leave dashboard → useEffect cleanup
   └─ clearInterval stops refresh cycle
   └─ Component unmounts

---

DOCTOR SIDE (Requesting Access)
─────────────────────────────────────────────────────────────────────────────
1. Doctor selects patient
   └─ Patient dashboard shows "37"

2. Doctor clicks "View Full Medical History"
   └─ FullMedicalHistoryModal opens
   └─ Consent verification screen shows

3. Doctor clicks "Start Verification"
   └─ API: GET /api/v1/consent/doctor/challenge/{patientId}
   └─ Response: {
        challengeId: "unique-id",
        options: ["82", "37", "19"]  // Shuffled, includes correct
      }
   └─ Display: 3 buttons

4. Doctor asks patient
   └─ "What's your verification code?"

5. Patient checks dashboard
   └─ Sees: "37"

6. Patient tells doctor
   └─ "My code is 37"

7. Doctor clicks "37"
   └─ API: POST /api/v1/consent/doctor/verify
   └─ Body: { challengeId: "...", selectedCode: "37" }

8. Backend validates
   └─ Check: Does "37" match stored code?
   └─ Check: Is code not expired?
   └─ Check: Are attempts < 3?
   └─ Result: ✅ Match!

9. Access granted
   └─ Modal shows: "✅ Access Granted"
   └─ Full medical history loads
   └─ Doctor can view complete patient data
