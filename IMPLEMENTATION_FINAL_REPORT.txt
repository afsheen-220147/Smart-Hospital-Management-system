🔧 API ENDPOINTS (FULL REFERENCE)
═══════════════════════════════════════════════════════════════════════════════

1. GET /api/v1/consent/patient/code/:patientId
   ──────────────────────────────────────────────
   Purpose: Patient retrieves their current verification code
   
   Parameters:
     - patientId (string, required): MongoDB patient ID or user ID
   
   Response: 200 OK
     { code: "XX" }  // 2-digit string from "00" to "99"
   
   Error: 400 Bad Request
     { error: "Patient ID required" }
   
   Behavior:
     • If patient has valid code (< 3 min old): return existing
     • If code expired or doesn't exist: generate new code
     • Code stored with 3-minute expiry timestamp
   
   Usage Example:
     fetch('/api/v1/consent/patient/code/user-123')
       .then(r => r.json())
       .then(d => console.log(d.code))  // "47"

───────────────────────────────────────────────────────────────────────────

2. GET /api/v1/consent/doctor/challenge/:patientId
   ──────────────────────────────────────────────────
   Purpose: Doctor requests 3-code challenge for patient verification
   
   Parameters:
     - patientId (string, required): The patient's ID
   
   Response: 200 OK
     {
       challengeId: "abc123xyz789",  // Unique challenge ID
       options: ["51", "64", "22"]   // Shuffled 2-digit codes
     }
   
   Error: 400 Bad Request
     { error: "Patient ID required" }
   
   Behavior:
     • Gets patient's current code (or generates new)
     • Creates array of 3 random 2-digit numbers
     • Includes patient's code in the array
     • Shuffles array so position is random
     • Creates challenge record with:
       - challengeId (unique)
       - correct code (stored securely)
       - 3 options (shuffled)
       - attempt counter (0)
       - expiry timestamp (3 minutes)
     • Returns challenge ID and options to doctor
   
   Note: Correct code is ALWAYS in the options array
   
   Usage Example:
     fetch('/api/v1/consent/doctor/challenge/patient-123')
       .then(r => r.json())
       .then(d => {
         console.log(d.challengeId)  // "xyz..."
         console.log(d.options)      // ["XX", "YY", "ZZ"]
       })

───────────────────────────────────────────────────────────────────────────

3. POST /api/v1/consent/doctor/verify
   ──────────────────────────────────
   Purpose: Doctor submits selected code for verification
   
   Request Body:
     {
       challengeId: "abc123xyz789",  // From challenge endpoint
       selectedCode: "22"             // Which button doctor clicked
     }
   
   Response: 200 OK (Valid Code)
     { 
       valid: true, 
       message: "Access granted" 
     }
   
   Response: 401 Unauthorized (Invalid Code)
     { 
       valid: false, 
       error: "Incorrect code" 
     }
   
   Response: 400 Bad Request (Missing Fields)
     { 
       error: "Challenge ID and code required" 
     }
   
   Response: 401 (Challenge Expired)
     { 
       valid: false, 
       error: "Challenge expired" 
     }
   
   Response: 401 (Max Attempts)
     { 
       valid: false, 
       error: "Max attempts exceeded" 
     }
   
   Behavior:
     • Find challenge by challengeId
     • Check if challenge exists: NO? → Error
     • Check if challenge expired: YES? → Delete & Error
     • Check if attempts >= 3: YES? → Delete & Error
     • Increment attempt counter
     • Compare selectedCode with stored correct code
     • If MATCH:
       ✅ Delete challenge (can't reuse)
       ✅ Return { valid: true }
     • If MISMATCH:
       ❌ Keep challenge (allow up to 3 tries)
       ❌ Return { valid: false, error: "Incorrect code" }
   
   Usage Example:
     fetch('/api/v1/consent/doctor/verify', {
       method: 'POST',
       headers: { 'Content-Type': 'application/json' },
       body: JSON.stringify({
         challengeId: 'abc123xyz789',
         selectedCode: '22'
       })
     })
     .then(r => r.json())
     .then(d => {
       if (d.valid) {
         console.log('✅ Access Granted')
         loadFullMedicalHistory()
       } else {
         console.error('❌', d.error)
       }
     })
